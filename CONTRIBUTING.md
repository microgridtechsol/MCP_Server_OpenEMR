# Contributing to OpenEMR MCP Server

Thank you for your interest in contributing! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/OpenEMR.git`
3. Install dependencies: `npm install`
4. Copy `.env.example` to `.env` and fill in your OpenEMR credentials
5. Start the dev server: `npm run dev`

## Making Changes

1. Create a feature branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test your changes locally
4. Commit with a clear message: `git commit -m "Add: description of change"`
5. Push to your fork: `git push origin feature/your-feature`
6. Open a Pull Request

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Follow the existing code style
- Do not commit `.env` files or any credentials
- Update the README if your change affects usage or configuration

## Reporting Issues

- Use GitHub Issues to report bugs or request features
- Include steps to reproduce for bugs
- Include your Node.js version and OS
